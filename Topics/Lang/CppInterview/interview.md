# C++八股

## C++基础

### 封装、继承和多态是什么？

**封装**：将具体实现过程和数据封装成一个类，只能通过接口进行访问，降低耦合性，使类成为一个具有内部数据的自我隐藏能力、功能独立的软件模块。

*意义：保护或防止代码在无意之中被破坏，保护类中的成员，不让类中以外的程序直接访问或者修改，只能通过提供的公共接口访问。*

**继承**：子类继承父类的特征和行为，复用了基类的全体数据和成员函数，具有从基类复制而来的数据成员和成员函数（基类私有成员可被继承，但是无法被访问），其中构造函数、析构函数、友元函数、静态数据成员、静态成员函数都不能被继承。基类中成员的访问方式只能决定派生类能否访问它们。增强了代码耦合性，当父类中的成员变量或者类本身被final关键字修饰时，修饰的类不能被继承，修饰的成员函数不能重写或修改。

*意义：基类的程序代码可以被派生类服用，提高了软件复用的效率，缩短了软件开发的周期。*

**多态**：不同继承类的对象对同一消息做出不同的响应，基类的指针指向或绑定到派生类的对象，使得基类指针呈现不同的表现形式。

*意义：对已存在的代码具有可替代性，对代码具有可扩充性，新增子类不会影响已存在类的各种性质，在程序中体现了灵活多样的操作，提高了使用效率，简化了对应用代码的编写和修改过程。*

### 多态的实现原理是什么？以及多态的优点？

多态分为动态多态和静态多态。其中**动态多态**是通过*虚函数*实现的，而**静态多态**则是通过函数重载（函数名相同，参数不同，两个函数在同一作用域），运算符重载，和重定义来实现的。

*优点：加强代码的可扩展性，可替换性，增强程序的灵活性，提高使用效率，简化对应用代码的编写和修改过程。*

### 虚函数是怎么实现的？

虚函数的实现原理基于**虚函数表**和**虚函数指针**。每个包含虚函数的类都会生成一个虚函数表，虚函数表是一个由指针构成的数组，每个指针指向一个虚函数的实现代码。在对象的内存布局中，编译器会在头部添加一个额外的指针（称为虚函数指针或虚表指针），这个指针指向该对象对应的虚函数表。当一个基类指针或引用调用虚函数时，编译器会使用虚表指针来查找该对象对应的虚函数表，并根据函数在虚函数表中的位置来调用正确的虚函数。虚函数在编译阶段生成，和普通函数一样存放在代码段，只是它的指针又存放在了虚函数表中。

### malloc内存分配的方式

malloc是c库函数，用于动态内存分配。调用malloc分配内存的时候会有两种方式向操作系统申请内存：

1. 当用户分配的内存小于128KB时，通过`brk()`系统调用从**堆**分配内存，将堆顶指针向高地址移动，获取内存空间，如果使用free释放空间，并不会将内存归还给操作系统，而是会缓存在malloc的内存池中，待下次使用。缺点是容易导致内存碎片的产生。
2. 当用户分配的内存大于128KB时，通过`mmap()`系统调用在文件映射区域（堆和栈之间）分配内存，free释放内存的时候，会把内存归还给操作系统，内存得到真正释放。缺点是容易引起缺页中断。

在申请内存的时候，会多分配16字节的内存，里面保存了内存块的详细信息，free会对传入的内存地址向左偏移16字节，然后分析出当前内存块的大小，就知道要释放多大的内存空间了。

### final标识符

修饰符final放在类后面表示该类无法被继承，放在函数后面表示该方法无法被重写（阻止虚函数的重载）。

### define和const的区别

处理阶段：define是在编译预处理阶段进行简单的文本替换，const是在编译阶段确定其值。

内存占用：define定义的宏常量，在程序中使用多少次就会进行多少次替换，内存中有多个备份，占用的是代码段的内存；const定义常量占用静态存储区域的空间，程序运行过程中只有一份。

### 编译的步骤是什么？

**预编译**：将头文件编译，进行宏替换，输出.i文件

**编译**：将其转化为汇编语言文件，主要做词法分析，语义分析以及检查错误，检查无误后将代码翻译成汇编语言，生成.s文件

**汇编**：汇编器将汇编语言文件翻译成机器语言，生成.o文件

**链接**：将目标文件和库链接到一起，生成可执行文件.exe

### 对象的生命周期

全局对象在main开始前被创建，main退出后被销毁。

静态对象在第一次进行作用域时被创建，在main退出后被销毁。

局部对象在进入作用域时被创建，在退出作用域时被销毁。

New创建的对象直到内存被释放的时候都存在。

### 为什么要进行内存对齐？内存对齐有什么好处？

内存对齐是处理器为了提高处理性能而对存取数据的起始地址所提出的一种要求。CPU访问内存的时候并不是逐个字节访问，而是以**字长**为单位访问，所以数据结构应该尽可能地在自然边界上对齐，如果访问未对齐内存，处理器需要做多次内存访问，而对齐的内存访问可以减少访问次数，提升性能。

*64位机器的字长为64位（8字节）。*

### 进程之间的通信方式有哪些？

**管道**：管道分为匿名管道和命名管道，管道本质上是一个内核中的一个缓存，当进程创建管道后会返回两个文件描述符，一个写入端一个输出端。缺点：半双工通信，一个管道只能一个进程写，一个进程读。不适合进程间频繁的交换数据

**消息队列**：可以边发边收，但是每个消息体都有最大长度限制，队列所包含的消息体的总数量也有上限并且在通信过程中存在用户态和内核态之间的数据拷贝问题

**共享内存**：解决了消息队列存在的内核态和用户态之间的数据拷贝问题。

**信号量**：本质上是一个计数器，当使用共享内存的通信方式时，如果有多个进程同时往共享内存中写入数据，有可能先写的进程的内容被其他进程覆盖了，信号量就用于实现进程间的互斥和同步PV操作不限于信号量+-1，而且可以任意加减正整数。

**信号**：用于异步事件通知（如`Ctrl+C`发送SIGINT终止进程），仅传递控制信息（如信号编号），不传输数据

**套接字**：支持跨网络通信，分为面向连接（TCP）和无连接（UDP），灵活但实现复杂。

### 线程之间的通信方式有哪些？

信号量、条件变量、互斥量

### IO多路复用有哪些？

select、poll、epoll都是IO多路复用的机制，可以监视多个文件描述符，一旦某个文件描述符进入读或写就绪状态，就能够通知系统进行相应的读写操作。

| 特性       | select                 | poll               | epoll             |
| ---------- | ---------------------- | ------------------ | ----------------- |
| 数据结构   | 固定大小位图（fd_set） | 动态数组（pollfd） | 红黑树 + 就绪链表 |
| 最大fd数量 | 1024                   | 无限制             | 无限制            |
| 时间复杂度 | O(n) 轮询              | O(n) 轮询          | O(1) 事件驱动     |
| 内存拷贝   | 每次调用全量拷贝       | 每次调用全量拷贝   | 首次注册后无拷贝  |
| 触发模式   | 水平触发（LT）         | 水平触发（LT）     | 支持 LT 和 ET     |
| 适用场景   | 低并发、简单场景       | 中低并发           | 高并发、海量连接  |

### epoll的底层实现是什么？

核心数据结构：

- 红黑树：存储所有注册的 fd 及其关注事件（`epoll_ctl` 添加），支持快速插入、删除和查找（O(log n)）
- 就绪链表：记录已就绪的 fd，通过回调机制直接添加就绪事件，避免遍历

工作流程：

	1. 初始化：`epoll_create` 创建 `eventpoll` 对象，包含红黑树和就绪链表
	1. 事件注册：`epoll_ctl` 将 fd 插入红黑树，并为每个 fd 注册回调函数 `ep_poll_callback`
	1. 事件等待：`epoll_wait` 检查就绪链表，若为空，进程挂起，超时或fd就绪时被唤醒；若链表非空，直接返回就绪事件
	1. 回调机制：当 fd 就绪时，内核调用 `ep_poll_callback`，将对应 `epitem` 加入就绪链表

核心优化：

- 事件驱动：仅关注活跃 fd，避免全量遍历
- 内存共享：通过 `mmap` 实现用户态和内核态共享就绪链表，减少数据拷贝

*同步与互斥：内核通过自旋锁或RCU（Read-Copy-Update）确保链表操作的原子性，避免用户态与内核态访问冲突。*

### mmap的机制是什么？有什么作用？

mmap（Memory Mapping）是一种将文件或设备直接映射到进程虚拟地址空间的机制，其核心目标是**避免用户态与内核态之间的数据拷贝**，并通过共享内存实现高效通信。

**虚拟地址映射**：mmap 通过 **页表机制** 将用户态虚拟地址与内核物理内存建立映射关系。在 64 位系统中，采用 **四级页表**（PGD、PUD、PMD、PTE）逐级索引物理页。

**动态页表分配**：系统不会预先分配所有页表，而是通过 **缺页中断** 动态创建页表项。首次访问虚拟地址时触发缺页中断，内核分配物理页并填充页表。

### 锁的底层原理是什么？

CAS机制：全称为Compare And Swap（比较相同再交换）可以将比较和交换操作转换为原子操作，CAS操作依赖于三个值：内存中的值V，旧的预估值X，要修改的新值B，如果旧的预估值X等于内存中的值V，就将新的值B保存在内存之中。（就是每一个线程从主内存复制一个变量副本后，进行操作，然后对其进行修改，修改完后，再刷新回主内存前。再取一次主内存的值，看拿到的主内存的新值与当初保存的快照值，是否一样，如果不一样，说明有其他线程修改，本次修改放弃，重试。）

### 原子操作的原理是什么？

原子操作是指不会被线程调度机制打断的操作，这种操作一旦开始，就一直运行到结束，中间不会有任何切换到另一个线程。

原理是：CPU提供了在指令执行期间对总线加锁的手段，CPU中有一根引线`#HLOCK pin`连接到北桥，如果汇编语言的程序在程序中的一条指令前面加上了前缀“LOCK”，经过汇编之后的机器码就使CPU在执行这条指令的时候把`#HLOCK pin`的电平拉低，这样别的CPU就暂时不能够通过总线访问内存了，保证了多处理器环境中的原子性。

### 死锁产生的条件，如何解决死锁？

产生死锁的四个条件：**互斥**条件、**请求和保持**条件、**不可剥夺**条件、**环路**等待条件。解决死锁的方法就是破坏上述任意一种条件。

### class与struct的区别

默认继承权限不同：class默认继承的是private继承，struct默认是public继承。

Class还可用于定义模板参数，关键字struct不能同于定义模板参数。

### 基类的构造函数和析构函数是否能为虚函数？

构造函数不能为虚函数，虚函数的调用是通过虚函数表来查找的，而虚函数表由类的实例化对象的vptr指针指向，该指针存放在对象的内部空间之中，需要调用构造函数完成初始化，如果构造函数为虚函数，那么调用构造函数就需要去寻找vptr，但此时vptr还没有完成初始化，导致无法构造对象。

析构函数可以为虚函数，但当使用父类指针指向子类时，只会调用父类的析构函数，子类的析构函数不会被调用，可能造成内存泄漏。

### 智能指针相关

使用普通指针，容易造成内存泄露（忘记释放）、二次释放、程序发生异常时内存泄露等问题等。C++11引入了智能指针，本质是一个封装了裸指针指针的类模板。

智能指针主要分为`shared_ptr`、`unique_ptr`和`weak_ptr`三种。

- `shared_ptr`：额外增加了**控制块**指针，开销是裸指针的两倍。控制块中包含了引用计数、弱引用计数、分配器、删除器和被管理对象的指针，当**引用计数**归零时调用析构函数释放资源。
- ` unique_ptr`：和裸指针开销几乎相同，禁止了拷贝语义，只能通过移动语义来转移所有权。
- `weak_ptr`：用于避免循环引用引起的死锁。影响弱引用计数。

当使用`make_shared`初始化时，控制块和实际对象通常分配在同一块内存中，减少了分配的次数，提高了性能。

### 匿名函数是什么？有什么作用？

匿名函数（Lambda表达式）本质上是一个重载了()符号的对象。使用匿名函数，可以免去函数的声明和定义。这样匿名函数仅在调用函数的时候才会创建函数对象，而调用结束后立即释放，所以匿名函数比非匿名函数更节省空间。

### （左值）引用和指针的区别？

指针可以不用初始化，引用必须初始化。

指针是一个变量，存放了内存中的一个地址；引用是为被引用的对象取一个别名。

指针占用8个字节空间，引用不额外占用空间。

*`++i`返回i本身，是一个左值；而`i++`返回的是i的值，是一个右值。*

### 右值引用是什么？有什么作用？

**右值引用**是为一个临时变量取别名，它只能绑定到一个临时变量或表达式（将亡值）上。

```cpp
int&& rr = 42;             // 正确：绑定到字面量
int&& rr2 = x;             // 错误：x 是左值
int&& rr3 = std::move(x);  // 正确：将左值转为右值
```

应用场景：

- **实现移动语义**：优化资源管理类（如 `std::vector`, `std::string`）。
- **容器操作**：`emplace_back` 直接构造元素，避免临时对象拷贝。
- **完美转发**：在泛型代码中保持参数的值类别。
